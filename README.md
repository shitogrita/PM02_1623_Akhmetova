# PM02_1623_Akhmetova

# Решение транспортной задачи методом северо-западного угла

Консольное приложение на C# для построения плана перевозок методом северо-западного угла.

## Возможности

Программа позволяет:

- вводить исходные данные вручную через консоль;
- загружать исходные данные из текстового файла;
- проверять баланс задачи: сумма запасов должна быть равна сумме потребностей;
- балансирует данные задачи при условиях несбалансированности;
- построить план перевозок методом северо-западного угла;
- вывести исходные данные, план перевозок и общую стоимость;
- сохранить результат в файл `result.txt`;

## Используемый метод

Метод северо-западного угла начинает заполнение плана с верхней левой ячейки матрицы перевозок.

На каждом шаге в текущую ячейку записывается минимальное значение из:

- остатка запаса текущего поставщика;
- остатка потребности текущего потребителя.

После этого поставщик или потребитель с нулевым остатком исключается из дальнейшего рассмотрения.

Метод формирует допустимый план перевозок, но не гарантирует минимальную общую стоимость среди всех возможных планов.

```text

```

## Ручной ввод

Сначала вводится количество поставщиков и потребителей.

Далее вводятся:

1. запасы всех поставщиков через пробел;
2. потребности всех потребителей через пробел;
3. строки матрицы стоимостей.

Пример:

```text
Количество поставщиков: 3
Количество потребителей: 3
Запасы через пробел: 20 30 25
Потребности через пробел: 10 25 40
Строка 1: 8 6 10
Строка 2: 9 7 4
Строка 3: 3 4 2
```

## Формат файла входных данных

Пример файла `input.txt`:

```text
3
3
20 30 25
10 25 40
8 6 10
9 7 4
3 4 2
```

Структура файла:

```text
количество поставщиков
количество потребителей
запасы поставщиков
потребности потребителей
строка матрицы стоимостей поставщика 1
строка матрицы стоимостей поставщика 2
...
```

## Пример результата

Общая стоимость:

```text
355
```
